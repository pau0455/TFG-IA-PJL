Orquestación con n8n

n8n es la herramienta de automatización que actúa como el sistema nervioso central de AITalk. Su arquitectura basada en nodos nos permite conectar APIs dispares sin necesidad de escribir código repetitivo.

Implementación en AITalk

Utilizamos el nodo Webhook como punto de entrada (Trigger). Cuando el usuario envía un mensaje desde el frontend, n8n recibe el payload HTTP POST y lo enruta al nodo de IA. n8n maneja lógica condicional: si el usuario pregunta por el menú, deriva el flujo a PostgreSQL; si quiere reservar, lo deriva a Google Calendar.

// Payload recibido por el Webhook
{
  "session_id": "usr_9841",
  "message": "Reserva para 4 a las 21:00"
}

Base de Datos (PostgreSQL)

PostgreSQL es vital para dar persistencia y contexto a la IA, permitiéndole conocer la "realidad" del negocio en lugar de depender de datos estáticos en el prompt.

Esquema y Consultas

La base de datos contiene esquemas para el inventario de menús (platos, alérgenos), información de locales (direcciones, horarios) y clientes. n8n ejecuta sentencias SQL dinámicas construidas según la intención detectada por Gemini.

SELECT plate_name, is_gluten_free
FROM menus
WHERE category = 'principales' AND is_gluten_free = TRUE;

API de Google Calendar

Empleamos la API REST de Google Calendar autenticada mediante OAuth 2.0 / Service Accounts en Google Cloud Console. Esto permite a AITalk leer la disponibilidad y escribir nuevos eventos.

Verificación y Creación

Antes de confirmar una reserva, n8n consulta la API para verificar solapamientos. Si hay hueco libre, se ejecuta el método POST para insertar el evento.

Motor LLM: Gemini 2.5 Pro

Utilizamos el modelo Gemini 2.5 Pro a través de Google AI Studio por su inmensa ventana de contexto y su avanzada capacidad de razonamiento lógico.

Function Calling (Uso de Herramientas)

La clave de nuestra integración no es solo generar texto, sino el Function Calling. Hemos instruido a Gemini con descripciones de funciones (ej. check_availability()). Cuando el usuario habla, Gemini responde solicitando a n8n que ejecute esa función específica en la BBDD.

Módulo de Audio: Whisper

Whisper es un modelo de reconocimiento automático de voz (ASR) open-source desarrollado por OpenAI. Destaca por su precisión y robustez frente a diferentes acentos y ruido de fondo.

Roadmap

La arquitectura de n8n está preparada para insertar un nodo que reciba archivos de audio (.ogg, .mp3), los envíe a una instancia local de Whisper para la transcripción Speech-to-Text, y pase el texto resultante al flujo estándar de Gemini.

Servidor Web Nginx

Nginx actúa como proxy inverso exponiendo de forma segura los webhooks locales de n8n al widget frontend y manejando los certificados SSL.
